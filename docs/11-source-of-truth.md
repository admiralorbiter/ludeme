# Source-of-Truth Documents

Maintain these documents as the project grows. Update them when direction changes, not just when features ship.

| Document | Purpose | When it matters most |
|---|---|---|
| `docs/` (this directory) | System architecture, stack, contracts, roadmap. | From day one. |
| `taxonomy/*.toml` | Canonical mechanic families, relation types, confidence levels. | Before the second demo. Update in version control only. |
| `FIDELITY.md` | Fidelity criteria reference with worked examples per level. | Before any demo is published. |
| `UX-FLOWS.md` | Primary journeys as annotated wireframes or written walkthroughs. | Before building each major surface. |
| `DECISION-LOG.md` | Why key product and schema decisions changed over time. | As soon as major tradeoffs begin. |
| `TEMPLATE-PACK.md` | Authoring templates for demos, mechanics, works, observations, experiments, collections. | Immediately, keep current with schema. |
| `RELEASE-RUBRIC.md` | What is publishable and what is still draft. Includes fidelity gate checklist. | Before broader public access or large batch imports. |
| `ROADMAP-BOARD` | Phase tracking, dependency map, deferred items. | Throughout build and content growth. |
